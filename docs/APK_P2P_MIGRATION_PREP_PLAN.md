# APK 内置 P2P 架构迁移准备与实施规划

## 1. 现存问题收口（先处理）

### 1.1 已完成修复
- 播放页返回时自动触发会话取消（避免后台继续下载）。
- 新增网关接口：`POST /play/session/{id}/cancel`。
- 取消会话时执行：
  - 取消 BT 任务。
  - 取消转码任务。
  - 清理 `runtime/streams` 下会话目录。
  - 从内存会话表移除 session。
- qBittorrent 取消策略改为 `delete_files=True`，退出页后同步删除下载数据。
- Flutter 播放页从 `WillPopScope` 升级为 `PopScope`，消除 deprecate issue。

### 1.2 待验证项（必须通过后再进入迁移开发）
- 验证返回播放页后网关日志出现：`POST /play/session/{id}/cancel`。
- 验证 qBittorrent 对应任务和数据文件被移除。
- 验证 `runtime/streams/<session_dir>` 被清理。
- 验证播放器退出后再次进入同剧集会创建新 session（无脏状态复用）。

### 1.3 结论
- 当前后端状态是“可用原型+联调可继续推进”，不是“生产级完善”。
- 可进入“迁移准备”阶段。

## 2. 迁移目标与边界

### 2.1 目标
- 不依赖独立后端服务。
- 在 APK 内部实现：
  - 种子/磁力解析。
  - P2P 下载调度。
  - 本地流式播放输出。
  - 会话生命周期（创建/状态/取消/重试）。

### 2.2 边界
- 迁移阶段优先 Android。
- 先完成本地 HTTP 回环流服务（`127.0.0.1`）；HTTPS 内置证书方案作为增强项。
- 保留现有 Flutter UI 层和状态页面，先替换数据通道，不先改 UI 交互模型。

## 3. 迁移准备工作规划

### 3.1 架构准备
- 定义统一 `Session Orchestrator` 接口，抽象当前网关 API：
  - `createSession`
  - `getSessionStatus`
  - `retrySession`
  - `cancelSession`
- 在 Flutter 保留现有 Repository，不改页面协议。
- 新增 `LocalGatewayAdapter`（in-app）与 `RemoteGatewayAdapter`（当前网关）双实现。

### 3.2 技术选型准备
- P2P 引擎：优先 `libtorrent`（JNI/FFI 封装）或 Android 原生可用 BT 库。
- 本地流服务：优先本地 HTTP（回环地址）；必要时再加 HTTPS 证书链。
- 播放路径：优先直读可播放文件分段；必要时走本地转码（ffmpeg-kit）。

### 3.3 系统能力准备（Android）
- 前台服务（下载与转码保活）。
- 电量与网络限制策略（Doze/后台限制）。
- 文件存储策略（分区存储目录与清理策略）。
- 通知与任务可见性（用户可中断下载）。

### 3.4 数据与可观测性准备
- 会话状态持久化（Room/SQLite）。
- 任务 crash 恢复策略（进程重启后恢复未完成任务或标记失败）。
- 统一错误码体系（BT、网络、解析、转码、播放）。

### 3.5 合规准备
- 版权与使用声明页。
- 用户显式确认与本地日志审计。
- 默认不开启公网入口，仅本机回环。

## 4. 迁移准备如何实现（实施路线）

### Phase A：收敛当前协议（1 周）
- 冻结会话协议字段和状态机。
- 完成退出清理与重试链路回归。
- 产出接口契约文档（字段/状态/错误码）。

验收标准：
- Flutter UI 不依赖具体后端实现。
- 现有网关回归通过，关键路径稳定。

### Phase B：客户端适配层落地（1 周）
- 新建 `GatewayAdapter` 抽象。
- `RemoteGatewayAdapter` 继续走当前网关。
- `LocalGatewayAdapter` 先用假实现打通调用链。

验收标准：
- 切换 Adapter 不改页面代码。
- 页面行为与当前一致。

### Phase C：内置 P2P 核心（2-3 周）
- 集成 BT 引擎（磁力/torrent 添加、文件列表、进度、取消、删除）。
- 实现顺序下载、首尾优先、可播阈值判定。
- 接入会话状态持久化。

验收标准：
- APK 内可独立创建和管理 BT 会话。
- 退出页可可靠取消并删除数据。

### Phase D：本地流服务与播放（2 周）
- 内置本地流服务（回环 HTTP）。
- 输出播放器可消费的 URL。
- 失败回退到重试/失败页。

验收标准：
- 无独立后端时可在 APK 内完整播放。
- 首帧时间与稳定性满足目标（例如首帧 < 15s）。

### Phase E：去独立后端切换（1 周）
- 默认切换到 `LocalGatewayAdapter`。
- 保留远程网关开关作为 debug fallback。
- 清理与下线 Python 网关依赖。

验收标准：
- 默认模式不需要启动本地 Python 服务。
- 核心链路：搜索/播放/退出清理可用。

## 5. 风险与应对
- 风险：P2P 低做种导致首帧慢。
  - 应对：顺序下载、首尾优先、候选源切换、超时重试。
- 风险：后台限制导致任务中断。
  - 应对：前台服务 + 可恢复任务模型。
- 风险：转码耗电高。
  - 应对：优先直读播放，转码只作兜底。

## 6. 当前执行顺序（建议）
1. 先完成“待验证项”回归并确认通过。
2. 进入 Phase A，冻结会话协议。
3. 再启动 Phase B 的适配层改造。
4. 最后进入内置 P2P 核心实现。
